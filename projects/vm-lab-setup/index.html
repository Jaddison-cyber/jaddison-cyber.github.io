Project 01 • Virtual Lab

Cybersecurity VM Lab Setup

This project documents the creation of a dedicated virtual machine environment for cybersecurity learning, OSINT workflow development, and safer experimentation outside of the host machine.

VirtualBox
Windows VM
OSINT Staging
Lab Documentation

Objective

Build a clean and isolated virtual lab environment that can support cybersecurity training, investigative tooling, browser hardening, and repeatable hands-on exercises.

Why It Matters

Separation matters. A dedicated VM gives me room to test, learn, break things, rebuild them, and document the process without mixing that activity into my main daily-use system.

Environment

  • Oracle VirtualBox for virtualization
  • Windows virtual machine environment
  • Initial browser hardening and OPSEC-focused setup
  • Preparation for OSINT and investigative tools

What I Did

I built out a separate virtual machine environment intended for cyber learning and investigative work. The goal was to create a more controlled space for experimentation while keeping my host machine cleaner and reducing operational sloppiness. Initial work included environment setup, early hardening decisions, and preparing the VM for future tools and workflow testing.

What I Learned

Even the early setup stage teaches a lot. Environment design, isolation, and documentation are not side quests. They are part of the mission. Small setup decisions can affect usability, repeatability, and overall security posture.

Next Steps

  • Refine browser OPSEC and privacy settings
  • Continue staging OSINT tools
  • Document workflow decisions more deeply
  • Use the VM as the base for future investigation writeups

Project Status

Active and expanding. This lab is the foundation layer for the rest of the portfolio and will continue to evolve as more tools, case notes, and investigative projects are added.
