Project 04 • Digital Forensics

Windows Event Log Forensics

This project documents a simulated insider threat investigation inside a Windows DFIR lab. The scenario focuses on unauthorized login attempts, successful access, process activity, and USB device insertion used to reconstruct a potential data exfiltration event.

Objective

Simulate a terminated employee attempting to access a system and use Windows event logs to determine whether unauthorized access and data exfiltration occurred.

Scenario

A user account belonging to a former employee attempted multiple logins using an outdated password. After several failures, authentication succeeded. The user then accessed files and connected a removable USB device before ending the session.

Key Evidence Identified

  • Event ID 4625 — Failed login attempts
  • Event ID 4624 — Successful authentication
  • Event ID 4688 — explorer.exe launched
  • Event ID 4688 — dllhost.exe activity
  • UserPnp Event ID 20003 — USB device insertion

Timeline Reconstruction

  • Multiple failed login attempts for the terminated account
  • Successful interactive login recorded
  • File Explorer opened by the user
  • File preview activity detected
  • USB device connected to the system
  • Potential file copy activity prior to logout

Tools Used

  • Windows Event Viewer
  • Security Event Logs
  • System Event Logs
  • VirtualBox DFIR Lab

What I Learned

Individual Windows events rarely tell the full story. Correlating authentication logs, process execution, and device insertion artifacts provides a clearer forensic timeline and strengthens investigative conclusions.

Project Status

Completed initial DFIR investigation scenario. Future updates will include screenshots of event evidence and expanded analysis of logout and account management events.
